Audio Artifact Detection: Deep Learning 3-Channel Architecture
This project leverages a Convolutional Neural Network (CNN) to detect digital audio compression artifacts, specifically targeting low-bitrate signals (8kbps). The system employs a multi-tier feature extraction approach using MFCC, Delta, and Delta-Delta coefficients to achieve high precision in distinguishing between original and degraded audio.

Key Features
3-Channel Feature Analysis: Audio data is represented as 3-channel tensors (similar to RGB images), allowing the CNN to learn complex temporal and spectral dynamics.

Deep Visual Inspection: Provides real-time Spectrograms and Delta/Delta-Delta feature heatmaps for forensic verification.

Real-time Prediction: Instant analysis for .wav and .mp3 files with an associated Artifact Confidence Score.

Professional UI/UX: A modern Streamlit dashboard designed for seamless workflow from input to deep analysis.

System Architecture
The pipeline consists of 5 core stages:

Preprocessing: Raw audio signal normalization and framing.

Feature Extraction: Extracting 13 MFCCs along with 1st (Delta) and 2nd (Delta-Delta) order derivatives.

Dataset Building: Data serialization into .npy format for efficient training.

CNN Training: Binary classification model (Clean vs. Artifact) optimized for spectral patterns.

Deployment: Web-based interface powered by Streamlit.

Installation & Usage
Clone the repository:
git clone https://github.com/thaiidduong/Audio-Artifact-Detection.git
cd Audio-Artifact-Detection

Install dependencies:
pip install -r requirements.txt

Training Pipeline:
python3 extract_mfcc.py
python3 build_dataset.py
python3 train.py

Launch Dashboard:
streamlit run app.py
